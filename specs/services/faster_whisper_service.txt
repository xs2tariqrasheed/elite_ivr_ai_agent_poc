Helper function for STT service.

Model tiny.en should be loaded before starting the app.

Base of faster_whisper pkg with locally running tiny.en model.

It should expect initial_prompt and hotwords args for different scenarios.

It should also expect the audio in (mono,16 kHz, float32 PCM, wav) in-memory format.

Transcripts may repeat short menu-style phrases (e.g. "new reservation, new reservation.") when initial_prompt or hotwords echo; intent training data should cover those forms so fastText is not confused.

e.g.

def transcribe_audio(audio, initial_prompt, hotwords):

    return segments
